Esta es una guía para la instalación de las herramientas que vamos a utilizar en este proyecto y sus dependencias. A continuación los pasos, cualquier duda o problema me lo pueden notificar.

❗Parte 1 - En esta parte realizaras las instalaciones de las herramientas básicas:

1️⃣ Primero descarga e instala Vscode desde un sitio web oficial (https://code.visualstudio.com/).

2️⃣Luego deberás instalar Node.js de igual forma, en su página oficial, en ella encontraras diferentes métodos relacionados con la instalación desde consola, no obstante más abajo puedes elegir instalar una versión copilada para tu versión de Windows. Ahora bien, si ya terminaste la instalación es recomendable revisar si Node.js ya instalo el gestor de paquetes NPM, normalmente al instalar node, este ya viene instalado por defecto, pero para revisar puedes hacer lo siguiente:

• Abre el Cmd de Windows (En el caso de que uses otro SO búscalo por internet), lo puedes abrir usando la combinación de teclas “Windows + R”, luego se abrirá la ventana de “Ejecutar”, en esa ventana escribirás “cmd” y luego le das a la tecla “Enter”.

• En este momento tendrás la Consola abierta, aquí escribirás node -v, el terminal te dirá tu versión de node, y posteriormente escribirás npm -v para verificar si tienes instalado “npm”.

❗Ahora no cierres el CMD, lo necesitaras luego.

3️⃣ En el siguiente paso Instalaras Git de igual forma en su página oficial de descarga (https://git-scm.com/downloads), descargaras el instalador y luego, en el apartado de “Choosing the default editor used by Git” seleccionaras “Use Visual Code as Git’s default editor”. Luego continúa con las opciones predeterminadas de la instalación.

❗Parte 2 - Ya con todo esto tendrás Instalado todas las herramientas necesarias para trabajar, ahora vamos con la configuración de estas.

1️⃣ Como primer paso tendrás que crearte una cuenta de GitHub, y esto es importantísimo por lo que viene a continuación, no cierres la pestaña.

2️⃣Ahora viene la configuración de Git, en el CMD de Windows, o en el bash de Git escribirás lo siguiente y presta atención, git config –global user.name “Tu Nombre y Apellido”, con este comando registraras tu nombre para los commit y ahora lo siguiente. Escribirás (Usando tu correo de GitHub) “git config –global user.email “Correo Primario o Correo Privado” ”

‼️ Con respecto a los correos (Importante leer este parrafo) : en GitHub, le darás click la parte superior derecha de la pestaña, donde aparece tu logo, y se te abrirá un menú, aquí seleccionaras la opción “Setting”. Luego en las opciones del costado izquierdo buscaras “Access > Emails” y seleccionaras esa opción. Aquí por defecto GitHub mantiene tu correo privado para los Push, puedes des enmarcar esta opción más abajo y utilizar tu correo con el que te registraste (El Correo Primario), o en el caso que no quieras, GitHub te da un correo especial (123456+ejemplo@ @users.noreply.github.com) que puedes usar como Correo Privado.

Ahora para verificar que todo se guardo, puedes escribir git config --list donde las variables user.name, user.email y core.editor reflejen los datos que insertaste, este último tiene que ser tu Editor de Texto, en este caso Vscode.

3️⃣Ahora las extensiones para Vscode y otras opcionales. Con el Vscode abierto, en el menú del lado izquierdo, seleccionaran la opción “Extensions”, también pueden usar Ctrl + Shift + X como atajo, y en esta sección es donde podrán seleccionar tus extensiones e instalarlas:

Obligatorias:

  • GitHub Pull Requests: Esta extensión te permitirá realizar lo
    Pull Request desde el Vscode sin morir en el intento. Una vez
    instalada tienes que vincular tu cuenta de GitHub en el icono
    “Accounts” en la esquina inferior izquierda del Vscode y seleccionar
    la opcion que hace referencia a GitHub.

Opcionales:

  • Error Lens: sobresalta los errores y advertencias en tu código.

  • Live Server: crea un servidor local que sirve tus archivos de
    proyecto. Esto te permite ver tu sitio web en un navegador sin
    necesidad de subirlo a un servidor web real. Agregado el 02/06/25

  • Live Server (Five Server): de igual manera que el Live Server,
    crea un servidor web local, pero este actualiza en tiempo real cuando
    se modifican documento CSS, y otras características nuevas(Leer
    Docu). Agregado el 02/06/25

Ya con esto tienes el entorno listo para trabajar, algunas extensiones se agregaran con el tiempo de ser necesario.
